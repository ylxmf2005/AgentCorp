# 复盘透镜

用这次 session 的形状要求的那几个透镜；每条观察都要引用它的 digest 数字或 entry index。每个透镜要求的是一个*改动*，不是一段描述——每个透镜结束时都落到"下次这里怎么省/怎么避"。

## Time

墙钟时间都花去哪了？按耗时给 turn 排序；把 agent 主动工作的时间和间隔（用户离开、配额等待、后台任务——按邻近 entry 给每段长间隔分类）分开。要找的是：耗时是同类阶段几倍的那些环节；没有依赖关系却串行做的活（本可以并行）；一次更省 cache、更友好的轮询或一条通知本可以省掉的等待；同一个文件被反复读了很多遍。

## Tokens

预算按经济价值算，花去哪了（output ≫ 新鲜 input ≫ cache-read，每 token 的成本依次递减）？按 output token 给 turn 排序；查 cache 命中率的走势（命中率崩掉说明上下文在churn，或者睡了太久、超过了 cache TTL）；找被重新推导的活——同一份证据、同一次搜索、同一段解释因为第一次没存下来或没被找到，被又做了一遍。子代理密集的 session：哪些 fan-out 值回了成本，哪些只是重复了主循环已经读过的东西？

## Friction

哪些尝试反复失败？错误爆发（一个 turn 里 ≥3 个错误形状的结果）以及是什么打断了这个循环；同一条命令以同样的形式反复失败（一条规则或一处缺失的检查，第一个 turn 就该拦住它）；权限拒绝和绕过它的方式；工具用错了（活儿不对口的工具，本有专门工具却调用了 N 次）；改完马上又被改一遍的编辑（锚点没对上、读的是过期内容）。

## Evolution signals（演进信号，→ `teamspace/skill-evolution/pending/`）

那些技能文本在这次 session 里失效的时刻：一条指令模糊到让人走错了一步，一条缺失的规则得有人临场发明，一个本该触发却没触发的 trigger，一个违背技能自身指引、结果反而更好的变通做法。每一处都变成一条带着失败轨迹（turn/entry 锚点）的提案——绝不在这里直接改技能。

## Project suggestions（项目建议，→ `CLAUDE.md`/`AGENTS.md`、`teamspace/compound/`，或发起人）

这次 session 对*仓库本身*学到了什么：一个耗费时间、该写进项目文档的陷阱；一条临场推导出来、该被记下来的惯例；一处这次 session 不得不反复绕开的重构或缺失测试；一份文档的缺失导致了绕路。

## Collaboration

人机协作的那道回路：本可以从仓库找到答案却问了人的问题（反过来也一样——本该问出口、却拿去瞎猜的地方）；拖慢了进度的关卡或确认，和真正拦住了错误的关卡或确认；不得不重复的指令（第一种措辞没说到位——换成什么措辞会更好）；发起人纠正方向的那一刻，以及更早的哪个信号本可以预示到它。
