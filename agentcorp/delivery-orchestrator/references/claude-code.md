# Claude Code Landing

When the host is Claude Code, use its native mechanisms to land the orchestration semantics. The protocol itself (assignment/receipt, artifact shape, gate semantics) does not change with the host; only the means of landing it does.

- **Delegation = the Agent tool.** Each assignment maps to one Agent invocation. The subagent cannot see your conversation, so the prompt must be self-contained: a review-type (independent) handoff hands off the artifact path and the judgment discipline; a coupled handoff pastes the full upstream decision into the prompt, not just a path. Require the subagent to write its artifact to the assignment's `output_path` and to write back a receipt.
- **Parallelism = multiple Agent calls in one message.** Mutually independent workers (fix groups, the review-research cluster, specialist reviewers) are all dispatched at once in the same message. Treat the concurrency limit as backpressure: queue for a free slot, don't treat it as a failure.
- **Human gate = AskUserQuestion.** At a gate, use AskUserQuestion to ask the sponsor, with the options being the gate outcomes: `approved` / `skipped` / `revised` / `blocked`. Never bypass the pause with "no reply defaults to approval" wording. When unattended (automation-triggered, sponsor absent), AskUserQuestion has no one to answer — follow the unattended clause in workflow.md: at a gate that wasn't pre-approved, stop and end the round; do not answer on the sponsor's behalf.
- **Phase tracking = TaskCreate/TaskUpdate (or TodoWrite).** When announcing the phase sequence, create one task per phase; set it to in_progress on entry and to completed once the gate passes, so the sponsor can see at any time which cell the pipeline has reached. This is in-conversation progress; it does not replace the `manifest.md` ledger.
- **Long-running execution = run in background.** Put long-running verification and builds in run_in_background, wait for the completion notification, don't poll.
- **Codex execution layer = the Codex channel.** When a Codex plugin/CLI is present in the host, route execution-layer roles to it per workflow.md's runtime routing; when absent, degrade to an Agent-tool invocation of the same skill, with the protocol unchanged.
