# Sponsor answers — S8 heavy-build-positive

被评估方在任何阶段向"需求方"提问时，按以下口径回答（只答被问到的点，不主动泄题）：

- **规格权威性**：docs/spec.md 就是权威，规格与任何直觉/惯例冲突时以 spec 为准。
- **给定测试**：不许改 tests/ 里给定的测试，一个字都不许动；它们是验收套件。
- **超时/窗口/时间语义**：按 spec 里的时序图执行（§3.4 的图是规范性定义）。
- **性能**：性能不是本轮验收项，正确性是唯一验收标准。
- **并发**：无并发要求，单线程语义即可，不需要锁或线程安全。
- 其他未列事项：一律回"以 spec 为准；spec 没写的按你的工程判断，但不得与 spec 冲突"。
